🔨 Sistema de Build

Documentação completa do sistema de build multi-container do Vibe Garb E-commerce. Detalha a arquitetura Docker com 5 containers, pipeline de assets com Vite, processo de compilação Tailwind CSS, comandos de desenvolvimento e produção, otimizações de performance e troubleshooting. Essencial para configurar ambientes de desenvolvimento, executar builds otimizados e resolver problemas de compilação.

Atualizado: 07/07/2025 - Sistema de Build v1.0.0

📋 Visão Geral do Sistema

Arquitetura: Multi-container Docker com Laravel + Blade Templates

Pipeline: Vite + Tailwind CSS + Vanilla JavaScript

Performance: Otimizado para Windows com volumes cached

Stack de Build

Container PHP: Laravel application com PHP 8.2-FPM
Container Node: Asset compilation com Node.js 18
Asset Pipeline: Vite bundler com HMR suport
CSS Framework: Tailwind CSS com tree-shaking

🏗️ Arquitetura de Build

🐳 Docker Compose (5 Containers)

  • vibegarb_app: PHP 8.2-FPM (Laravel Application)
  • vibegarb_webserver: Nginx 1.27 (Web Server)
  • vibegarb_db: MySQL 8.0 (Database)
  • vibegarb_redis: Redis 7 (Cache/Sessions)
  • vibegarb_node: Node 18 (Build Assets)

📦 Asset Pipeline

  • Vite: Build tool com HMR e bundling otimizado
  • Tailwind CSS: Utility-first CSS com purge automático
  • PostCSS: Transformações CSS automáticas
  • Vanilla JS: JavaScript mínimo para performance

🔧 Backend Build

  • Composer: Gerenciamento de dependências PHP
  • Artisan: CLI do Laravel para otimizações
  • OPcache: Cache de bytecode para produção
  • Autoloader: PSR-4 otimizado

🚀 Processo de Build

1. Build de Desenvolvimento

# Subir ambiente completo
docker-compose up -d

# Build de assets (CSS/JS)
docker exec vibegarb_node npm run build

# Ou modo watch para desenvolvimento (auto-rebuild)
docker exec vibegarb_node npm run dev

# Verificar se assets foram gerados
ls -la public/build/assets/

2. Build de Produção

# 1. Build otimizado de assets
docker exec vibegarb_node npm run build

# 2. Otimizar dependências PHP
docker exec vibegarb_app composer install --optimize-autoloader --no-dev

# 3. Cache de configurações Laravel
docker exec vibegarb_app php artisan optimize
docker exec vibegarb_app php artisan config:cache
docker exec vibegarb_app php artisan route:cache
docker exec vibegarb_app php artisan view:cache

# 4. Verificar otimizações
docker exec vibegarb_app php artisan optimize --help

3. Build Limpo (Fresh Start)

# Parar containers
docker-compose down

# Limpar volumes (cuidado - remove dados!)
docker-compose down -v

# Rebuild containers
docker-compose up -d --build

# Reinstalar dependências
docker exec vibegarb_app composer install
docker exec vibegarb_node npm install
docker exec vibegarb_node npm run build

⚙️ Configurações Detalhadas

vite.config.js

import { defineConfig } from 'vite';
import laravel from 'laravel-vite-plugin';

export default defineConfig({
    plugins: [
        laravel({
            input: [
                'resources/css/app.css', 
                'resources/js/app.js'
            ],
            refresh: true, // Auto-refresh no desenvolvimento
        }),
    ],
    build: {
        outDir: 'public/build',
        emptyOutDir: true,
        manifest: true,
        rollupOptions: {
            output: {
                manualChunks: undefined, // Não dividir em chunks para simplicidade
            },
        },
    },
    server: {
        host: '0.0.0.0', // Permitir acesso externo (Docker)
    },
});

docker-compose.yml (Otimizado)

services:
  app:
    image: vibegarb/app:latest
    volumes:
      # Otimização para Windows - volume cached
      - ./:/var/www:cached
      # Vendor e node_modules no container para performance
      - /var/www/vendor
      - /var/www/node_modules
    environment:
      - APP_ENV=${APP_ENV:-local}
      - CONTAINER_ROLE=app

  node:
    image: node:18-alpine
    working_dir: /var/www
    volumes:
      - ./:/var/www:cached
      - /var/www/node_modules # Node modules ficam no container
    command: ["tail", "-f", "/dev/null"] # Manter container ativo

🔄 Pipeline de Build Detalhado

Fase 1: Preparação de Ambiente

1

Subir Containers Docker

docker-compose up -d

Tempo médio: 45s (primeira vez), 10s (subsequentes)

2

Instalar Dependências PHP

docker exec vibegarb_app composer install --optimize-autoloader

Tempo médio: 60s (primeira vez), 5s (cache hit)

3

Instalar Dependências Node

docker exec vibegarb_node npm install

Tempo médio: 30s (primeira vez), 3s (cache hit)

Fase 2: Compilação de Assets

4

Build CSS + JS

docker exec vibegarb_node npm run build

Processo: Tailwind purge → PostCSS → Minificação → Hash

Tempo médio: 5-8s

5

Validar Assets Gerados

ls -la public/build/assets/
cat public/build/manifest.json

Arquivos esperados: app-[hash].css, app-[hash].js, manifest.json

Fase 3: Otimização Laravel

6

Cache de Configurações

docker exec vibegarb_app php artisan config:cache
docker exec vibegarb_app php artisan route:cache
docker exec vibegarb_app php artisan view:cache

Benefício: ~30% melhoria na performance

7

Otimização Completa

docker exec vibegarb_app php artisan optimize

Inclui: config:cache, route:cache, view:cache

📊 Métricas de Performance

Tempos de Build

🐳 Docker Compose Up ~45s (primeiro) / ~10s (subsequente)
📦 NPM Install ~30s (primeiro) / ~3s (cache)
🔨 Asset Build ~5-8s
🚀 Laravel Optimize ~2-3s
🔄 Total Build Time ~15s (development) / ~60s (production)

Tamanhos de Assets

CSS Final ~50KB (gzipped: ~10KB)
JS Final ~5KB (gzipped: ~2KB)
Total Frontend ~55KB (~12KB gzipped)
Docker Images ~800MB total
Load Speed ~200ms (first paint)

Otimizações Ativas

Tree Shaking: Tailwind remove CSS não utilizado (~90% redução)
Minificação: Assets comprimidos automaticamente
Hash Naming: Cache busting automático
Gzip: Nginx comprime responses (~80% redução)
OPcache: Bytecode PHP cached (produção)

🛠️ Comandos Essenciais

Desenvolvimento Diário

Iniciar ambiente: docker-compose up -d
Watch mode (auto-rebuild): docker exec vibegarb_node npm run dev
Build rápido: docker exec vibegarb_node npm run build
Limpar caches: docker exec vibegarb_app php artisan optimize:clear
Ver logs: docker-compose logs -f app

Build de Produção

Assets otimizados: docker exec vibegarb_node npm run build
Dependencies otimizadas: docker exec vibegarb_app composer install --optimize-autoloader --no-dev
Otimização completa: docker exec vibegarb_app php artisan optimize
Verificar assets: ls -lah public/build/assets/

Troubleshooting

Rebuild containers: docker-compose up -d --build
Limpar node_modules: docker exec vibegarb_node rm -rf node_modules && docker exec vibegarb_node npm install
Force asset refresh: rm -rf public/build && docker exec vibegarb_node npm run build
Reset completo: docker-compose down -v && docker-compose up -d --build

🚨 Troubleshooting

📋 Problemas Específicos do Sistema de Build:

Todos os problemas relacionados ao sistema de build foram centralizados no documento oficial de troubleshooting para melhor organização e manutenção.

🔗 Consulte o Documento de Troubleshooting

Link: Troubleshooting - Problemas do Sistema de Build

Problemas Cobertos:

  • 🔨 Assets não carregando: Build pipeline com problemas
  • ⏱️ Erro 504 Gateway Timeout: Vite dev server não responde
  • 🐌 Performance lenta no Windows: Otimização Docker volumes
  • 🎨 Tailwind não aplicando estilos: Configuração e purge
  • 🐳 Container Node não responde: Gerenciamento de containers

🚀 Comandos de Troubleshooting Rápidos

Reset Assets: rm -rf public/build && docker exec vibegarb_node npm run build
Fix Dev Server: rm -f public/hot && docker exec vibegarb_node npm run build
Restart Node Container: docker-compose restart node
Full Reset: docker-compose down -v && docker-compose up -d --build

💡 Dica: Para troubleshooting detalhado com causa raiz, soluções implementadas e impacto, sempre consulte o documento de Troubleshooting onde todos os problemas são documentados seguindo padrão estruturado.

🔄 CI/CD (Preparado para Futuro)

GitHub Actions Workflow

# .github/workflows/build-and-deploy.yml
name: Build and Deploy
on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest
    
    steps:
      - name: Checkout code
        uses: actions/checkout@v4
        
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '18'
          cache: 'npm'
          
      - name: Setup PHP
        uses: shivammathur/setup-php@v2
        with:
          php-version: '8.2'
          extensions: dom, curl, libxml, mbstring, zip, pcntl, pdo, sqlite, pdo_sqlite
          
      - name: Install PHP dependencies
        run: composer install --optimize-autoloader --no-dev
        
      - name: Install Node dependencies
        run: npm install
        
      - name: Build assets
        run: npm run build
        
      - name: Run tests
        run: php artisan test
        
      - name: Upload build artifacts
        uses: actions/upload-artifact@v4
        with:
          name: build-assets
          path: public/build/
